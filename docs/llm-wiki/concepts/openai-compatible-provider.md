# OpenAI-compatible provider

**Purpose** Chat with any OpenAI-compatible Chat Completions provider as a full agent — streamed text and reasoning, ClawDEA's tools through the existing approval/review paths, cancel-and-continue steering, profile-native session resume, and configured per-token pricing — driven entirely by an importable JSON **profile** with no provider-specific code and no bundled private presets.

## Invariants

- **Profiles carry no secrets.** A profile is a JSON document (base URL, endpoints, headers, model mapping, model capability rules, credential-flow schema, pricing). Credentials live only in the IDE's `PasswordSafe`, keyed per profile ID; template/configured exports strip secrets ([ProfileImportExport.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/profile/ProfileImportExport.kt), [ProfileCredentialStore.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/auth/ProfileCredentialStore.kt)).
- **Capability gates agentic chat.** `ModelCapabilityResolver.resolve` picks with precedence `userOverride > endpointCapability > matchingRule > COMPLETION_ONLY`. Only `AGENTIC` may start agentic chat; `COMPLETION_ONLY` and `UNKNOWN` cannot. The factory enforces this before constructing a live backend ([ModelCapabilityResolver.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/catalog/ModelCapabilityResolver.kt), [AgentBackendFactory.kt](../../../src/main/kotlin/com/adobe/clawdea/cli/backend/AgentBackendFactory.kt)).
- **Verification is explicit and never automatic.** `ModelCapabilityVerifier` sends ONE request with a single harmless no-op probe function and no project tools; it returns `AGENTIC` only when the model calls that exact function with valid-JSON arguments, `COMPLETION_ONLY` otherwise, and `UNKNOWN` on transport failure (never silently agentic). Wired behind the settings card's **Verify Tool Support** button so no hidden token usage is incurred ([ModelCapabilityVerifier.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/catalog/ModelCapabilityVerifier.kt)).
- **Exactly-once tool execution.** The agent loop tracks `completedToolCallIds`; a tool-call ID already completed is never executed again — including after a resume that rebuilds state from the ledger ([AgentLoopController.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/agent/AgentLoopController.kt), [ConversationState.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/agent/ConversationState.kt)).
- **Steering preserves valid text, discards incomplete tool calls.** Cancel-and-continue cancels the running round, persists any valid partial assistant text as context, drops the round's incomplete tool-call fragments (they live in a local assembler that never reached `completed()`), and continues with the new user message — without emitting a terminal result for the cancelled round ([OpenAiCompatibleAgentBackend.kt](../../../src/main/kotlin/com/adobe/clawdea/cli/backend/OpenAiCompatibleAgentBackend.kt), [SteeringController.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/agent/SteeringController.kt)).
- **Remote errors never trigger a silent provider fallback.** A cross-provider handoff involving this backend is user-confirmed via `ProviderFallbackPrompt`; only an affirmative result replays the transcript (text-only) under the alternate provider ([ProviderFallbackPrompt.kt](../../../src/main/kotlin/com/adobe/clawdea/chat/ProviderFallbackPrompt.kt), [ChatSession.kt](../../../src/main/kotlin/com/adobe/clawdea/chat/ChatSession.kt)).
- **Session ledgers live under the user's ClawDEA data dir, never the repo.** Append-only JSONL at `~/.clawdea/sessions/<sha256(project-path)>/<session-id>.jsonl`, written owner-only where the OS supports it ([OpenAiSessionLedger.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/session/OpenAiSessionLedger.kt)).

## Turn pipeline

1. `AgentBackendFactory.create` resolves the profile, credential, and selected model, checks capability, builds the MCP tool catalog + approval gate, and constructs `OpenAiCompatibleAgentBackend` (or an error backend carrying a readiness message). It never falls through to a Claude/Codex process ([AgentBackendFactory.kt](../../../src/main/kotlin/com/adobe/clawdea/cli/backend/AgentBackendFactory.kt)).
2. On `sendMessage`, the backend runs an orchestration loop: each round is a lazily-started child job so mid-turn steering can cancel just that round. `AgentLoopController.runTurn` streams one completion, assembles tool calls, executes them exactly-once through the tool executor, and repeats until no tool calls remain (bounded by tool-round, wall-clock, and context limits).
3. The HTTP client streams `POST <base-url><chatCompletions>` with `stream: true`; `AgentChatSseParser` turns SSE lines into text / reasoning / tool-fragment / usage / finished / failure events ([OpenAiCompatibleClient.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/client/OpenAiCompatibleClient.kt), [AgentChatSseParser.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/agent/AgentChatSseParser.kt)).
4. On a stream failure the backend applies `AgentRetryPolicy`: 401/403 → renew credential once; 429 → wait `Retry-After` (bounded, ≤2 attempts); pre-output 5xx/connection → exponential backoff (≤2 attempts); any failure after partial output → ask the user; otherwise fail. Terminal states emit exactly one `CliEvent.Result` ([AgentRetryPolicy.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/agent/AgentRetryPolicy.kt)).
5. Every user/assistant/tool_use/tool_result/reasoning/usage step is appended to the session ledger. Resume rebuilds `ConversationState` (messages + `completedToolCallIds`) from the ledger; same-profile resume is native, cross-profile/backend resume replays transcript text as context ([OpenAiSessionScanner.kt](../../../src/main/kotlin/com/adobe/clawdea/provider/openai/session/OpenAiSessionScanner.kt)).
6. Usage is priced from the profile's configured per-token rates (input/output/cached/reasoning) and attributed per profile.

## Anti-patterns

- **Starting agentic chat on an UNKNOWN or COMPLETION_ONLY model** — the factory returns a readiness error instead. Resolve or verify capability first.
- **Auto-running capability verification or probing on selection** — verification must be user-initiated (the button), or it silently spends tokens.
- **Silently continuing a conversation under a different provider** — always route cross-provider handoff through `ProviderFallbackPrompt`; tool-protocol state is intentionally dropped in the text-only replay.
- **Re-executing a tool call on resume** — consult `completedToolCallIds`; a completed ID must return its recorded result, not run again.
- **Writing session data into the repo** — ledgers belong under `~/.clawdea/sessions/`, never the project tree.
- **Putting provider-specific names, hosts, or model IDs in code/resources/docs** — the feature is generic; private profile details are imported at runtime only.
